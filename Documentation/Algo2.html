Algorithm - 2 (Knuth's algorithm)

This algorithm ensures that aesthetically, the left node stays left, and the right node stays right, (relative to the parent) and therefore is created for binary trees. It also follows the first aesthetic specification, that all nodes of the same level lie on the same line [height] in the illustration, provided the correct height of each node is contained within the structure.

Methodology

The coordinate assignment involved follows inorder traversal of the tree. This is done so that the left edge can be reached first and x coordinates can be assigned based on this reference. A variable keeps track of the next number in the inorder sequence [left child, node, and then right child].

Further, each node also has a status field in its structure, which is used to check which node is being numbered at a given instance.

The status field is encoded as:
  • First Visit, before the node has actually been visited, as status = 1,
  • Left Visit, when the left child is being numbered, as status = 2
  • Right Visit, when the right child is being numbered, as status = 3

The algorithm assumes that each node on the binary tree, has been given a correct height. This will be relied upon to ensure that the drawing abides by the first aesthetic specification.

Analysis

Even though the left node stays left and the right node stays right of the parent, strictly following the aesthetic requirement may cause the resultant tree may be too wide.
